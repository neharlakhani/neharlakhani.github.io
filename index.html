FinSight - Enhancing Financial Analytics through
Multi-Modal Data and a Mixed RAG Approach
1 Motivation and Objectives
In today's fast-paced ﬁnancial markets, large institutions such as hedge funds and investment
ﬁrms have access to vast amounts of real-time ﬁnancial data, cutting-edge technology, and
sophisticated AI models. These resources allow them to predict market movements, optimize
investment strategies, and generate substantial returns. Their ability to leverage data, models,
and human expertise places them at a signiﬁcant advantage.
However, individual investors and small ﬁrms often lack access to such advanced ﬁnancial tools
and infrastructures. The challenge they face is in gathering and analyzing vast datasets,
integrating relevant ﬁnancial news, and using this information to make informed investment
decisions. This gap creates a divide between institutional and retail investors, where smaller
players struggle to stay competitive.
The objective of this project is to democratize stock recommendation systems by developing a
powerful tool that integrates Retrieval-Augmented Generation (RAG) and Large Language
Models (LLMs) to provide personalized stock insights using real-time multi-modal data. The
tool will offer dynamic predictions and recommendations, enabling users to stay competitive
without the need for large-scale data infrastructures.
2 Related work
Signiﬁcant advancements have been made to create a one-stop platform that streamlines
access to critical ﬁnancial intelligence. BloombergGPT [1] is a 50 billion parameter LLM
trained on a large variety of ﬁnancial data. It greatly outperforms the other general LLMs in
ﬁnancial tasks while remaining competitive in general benchmarks as well. However, the
domain-speciﬁc dataset that it is trained on, represents less than 50% of the overall training
data thereby limiting its ability to perform all ﬁnance-speciﬁc tasks. Additionally, models like
Bloomberg and FinMA [2] are not up-to-date with the latest ﬁnancial market as they have
been pre-trained on static datasets making them quickly outdated in the fast-moving ﬁnancial
world.
FinGPT[3] addresses this by providing access to real-time, diverse, and high-volume datasets
through web scraping. However, this can lead to noise in the data and potential legal or ethical
concerns regarding the use of scraped information . Web scraping and public APIs are not the
most reliable sources of accurate information which might face restrictions in the future.
The above drawback is overcome in the Stock-Chain framework [4]. This work ﬁnetunes the
StockGPT[5] model on AlphaFin dataset and uses Retrieval Augmented Generation (RAG) to
generate real-time and accurate ﬁnancial content. The system retrieves ﬁnancial documents
relevant to a user’s query from a vector database that contains knowledge about stock market
data, reports and ﬁnancial news. The vector database is updated daily with the latest domain
knowledge. However, despite its higher accuracy, the multi-step nature of the Stock-Chain
framework increases its computational complexity, making it resource-intensive and
potentially impractical for smaller organizations lacking signiﬁcant hardware or cloud
infrastructure.
The proposed solution will build upon the existing Stock-Chain framework, enhancing both its
performance and output quality.
3 Proposed work
Proposed Architecture:
The main goal of this project is to address key limitations in the StockChain framework
addressed in the paper [4]. We believe the limitations are limited diversity of datasets and a
restriction of using only a vector based similarity search in the RAG phase, we aim to address
the limitations using the following techniques
1. Increasing diversity of datasets by leveraging multi-modal data - While AlphaFin
combines traditional research datasets, real time ﬁnancial analytics and chain of
thought prompting, the data diversity is still limited due to the use of purely textual
information from news articles, reports etc. The quality of ﬁne tuning can be enhanced
by adding multi-modality in the data such as audio/video interviews from companies,
analyst presentations and graphical insights from technical charts. We believe this will
increase the adaptability of the stock chain framework to private and institutional
investors since they can now add their own analysis to better ﬁne tune the model. This
can also be continually enhanced through an online learning based mechanism which
captures the data in real time from sources of varying modality (this is yet to be
explored)
2. Integration of knowledge graph (a mixedRAG approach) - To further improve the RAG
component of the stock-chain framework we can integrate a knowledge graph which
captures entities, relationships and attributes relevant to ﬁnancial analytics. We
believe that with this integration the Stock-chain framework can leverage structured
relationships and attributes to provide more accurate and comprehensive ﬁnancial
analytics. The knowledge graph building process can be split into the following
components
a. Deﬁne the graph structure - Add a graph structure with nodes representing
key entities such as Financial Metrics, Companies, Market Trends, Events etc.
and establish edges to represent relationships between these entities such as
node A (Event) -> “Affects” -> (Financial Metric) node B.
b. Specify Node Attributes (for each type of node) - The nodes and edges should
be represented with key attributes relevant to ﬁnancial analytics and capture
relationships between different entities
c. Populate the graph - We can collect data from several sources to build this
knowledge graph. Can leverage natural language techniques to establish the
relationships
d. Integrate with the RAG system - Modify the underlying RAG system to also
query the knowledge graph along with the vector similarity search.
e. Continuous Update - Implement a mechanism to update this graph as new data
becomes available (can use the same sources as the vector db update)

5 Evaluation and Testing Method
The main goals of the evaluation process is to check the correctness of FinSight’s outputs, test
for overﬁtting, generalizability and benchmark its performance against state-of-the-art
ﬁnancial analysis models.
Given relevant context, the model is expected to predict return direction (up or down) for a
given stock at time t - the ﬁrst round of evaluation will be checking the accuracy, precision,
F1-score, recall and other metrics commonly used for binary classiﬁcation tasks.
To evaluate the model’s effectiveness as a personal ﬁnancial advisor, the model's return
direction predictions can be used to inform portfolio trading decisions. The model's
robustness to diverse ﬁnancial analysis tasks can be tested by applying a variety of portfolio
construction approaches, including different weighing strategies, rebalancing at various
intervals (daily, monthly, quarterly), and diversifying across asset classes, market
capitalizations, and sectors. The model's predictions can further be stress-tested across
out-of-sample periods of ﬁnancial turbulence to test robustness to overﬁtting and its
resilience to market shocks.
Trading strategies can also be built on the predictions from other state-of-the-art ML, DL and
open-sourced ﬁnancial-LLM models like XGBoost, LSTMs, FinGPT to benchmark FinSight's
performance. Metrics including Annualized Rate of Return, Accuracy Rate, Volatility, Sharpe
Ratio, Calmar Ratio, Maximum Drawdown, Alpha, Beta, Value-At-Risk can be compared
across the resultant portfolios to demonstrate the potential outperformance of FinSight's
predictions. We expect FinSight to outperform in portfolios with greater recency.Additionally,
for any ﬁnancial analysis Q&A tasks, the ROUGE score can be used to determine quality of the
response.
